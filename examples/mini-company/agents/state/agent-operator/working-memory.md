# Working Memory — The Operator

## The World

TaskFlow is Day 1. The Builder has a task queued to create a landing page. Once that's done, I verify it. That's the first operational cycle: build → verify.

## What Matters Now

Nothing to verify yet — waiting on the Builder. When the landing page ships, I'll check:
- Does the HTML file exist at the expected path?
- Is it valid HTML (opens and closes tags properly)?
- Does it contain real content (not lorem ipsum or TODOs)?
- Is the file size reasonable (not empty, not bloated)?

## What I'm Watching

- The task queue — are tasks moving through, or are they getting stuck?
- The filesystem — is it clean and organized, or is clutter accumulating?
- Build outputs — do they exist and do they work?
